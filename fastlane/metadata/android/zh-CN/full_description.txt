XFiles 是一款离线的 Android 文件管理器，沿用 X-plore 的操作方式：两栏互不干扰、文件夹原地展开成树、压缩包可以像文件夹一样直接走进去。

它没有 INTERNET 权限。Android 系统本身就会阻止它建立任何网络连接，所以你浏览、复制、打开的任何东西都不可能离开这台设备。没有统计埋点，没有崩溃上报，没有广告，没有账号，没有云。

双栏树形浏览
• 两栏彼此独立 —— 平板和横屏下左右并排，手机上是可滑动切换的分页
• 文件夹原地展开，带缩进引导线，而不是一层一层地把当前视图替换掉
• 图片缩略图和视频首帧直接显示在树里，抽取一次后落盘缓存，下次立刻出图
• 跨栏复制/移动，目标目录由一个明确的选择器决定，默认就是另一栏

文件操作
• 多选、复制、移动、重命名、删除、新建文件夹
• 冲突时可选 跳过 / 覆盖 / 两个都留
• 耗时任务放进前台服务继续跑，带进度和取消按钮

压缩包当文件夹
• zip、jar、apk、7z、tar(.gz/.bz2/.xz)、rar 都能原地浏览，不用先解压
• 打包时用满所有 CPU 核心
• 想解压就复制到你选定的目录

内置查看器
• 图片查看器，支持翻页和双指缩放
• 文本查看器，可编辑保存
• 十六进制查看器，大文件按需分页
• 音频播放器
• 视频播放器支持逐帧精确定位 —— 点一下时间读数变成帧计数，可以一帧一帧地走，按帧拖动并实时预览

应用管理
• 已安装和系统应用，带真实图标、版本号和包名
• 每个应用的 activity / service / receiver / provider 都能展开成树，并显示 exported 和 enabled 状态
• 每个应用的 base.apk 和各个 split APK 都列出来，而且能继续展开，毕竟 APK 本来就是个 zip
• 支持安装、启动、卸载，或者把 APK 复制出来当文件分享

ROOT 访问（可选，默认关闭）
• 在设置里打开，即可在已 root 的设备上以超级用户身份浏览整个文件系统
• 另有一个独立的只读开关，会挡掉所有写操作，让你能看但不会把系统改坏
• 拿不到 su 时退化为只读视图

现代 Android
• Material 3 Expressive，支持动态取色、浅色/深色/跟随系统
• 真正的边到边，没有顶部 app bar
• 隐藏文件、文件夹优先、排序等选项

关于"所有文件访问权限"
XFiles 申请"所有文件访问权限"，是因为通用文件管理器本就需要这个：它必须能列出、打开、修改共享存储里任意位置的文件，包括不是它自己创建的目录。这个权限不作他用。而且本应用没有网络权限，根本没有办法把你的文件发到任何地方。
